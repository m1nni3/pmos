PMOS Financials — Design Decisions

Comparing approaches for budget data, routing, and dashboard charts. All functionality preserved regardless of choice.

1. Budget Data Strategy

The P&L sub-page needs a budget number per property to compare against actuals. Two ways to get it:

🗄️ D1 budgets table most flexible

New table: { id, property_id, month, year, budget_amount, category }

New API: GET /api/budgets?property_id=X&year=2026

Budget categories: Rental income, Levy income, Management fees, Maintenance, Utilities, Insurance, Municipal costs

Seed data: D1 migration with budget rows — budget = current actual × 1.05 (5% growth assumption), then adjustable

ProCon
✓ Exact budget figures✗ Must write & apply D1 migration
✓ Per-category breakdowns✗ Needs UI to add/edit budgets
✓ Month-by-month variance✗ Seed data can get stale
✓ Audit-ready for investors✗ ~45 min to implement fully
med effort
🧮 On-the-fly calculation fastest to ship

Formula: Budget = purchase_price × 0.06 ÷ 12 (6% annual yield target)

Alt formula: Budget = gross_rental × 1.10 (10% above current rental)

Where it lives: In the Worker API — a computed column on /api/properties or a new /api/budgets/calculate endpoint

No DB changes, no migrations, no seed data.

ProCon
✓ Zero database changes✗ Rough estimate only
✓ Ship in ~15 minutes✗ No per-category breakdown
✓ Always up-to-date (based on real purchase price)✗ Same % for every property
✓ Easy to swap for table later✗ Can't reflect seasonal variance
low effort

2. Routing Approach

How the 4 finance sub-pages (Dashboard, Ledger Hub, P&L, Maintenance) navigate within the Financials page.

📦 Internal state (detailView) current pattern
How it works
setDetailView('ledgers') renders ledger content, setDetailView(null) returns to dashboard. URL stays at /financials.
ProCon
✓ Already implemented — zero new code✗ Refresh resets to dashboard
✓ No changes to App.jsx or routes✗ Browser back/forward doesn't navigate sub-pages
✗ Can't deep-link (e.g., /financials#ledgers)
✗ Can't link from PropertyDetail → Financials/Ledgers
🔗 Sub-routes (/financials/*) recommended
How it works
Routes: /financials → Dashboard, /financials/ledgers → Ledger Hub, /financials/pnl → P&L, /financials/maintenance → Maintenance
ProCon
✓ Refresh preserves sub-page✗ ~20 lines: new Routes in App.jsx + useParams in Financials
✓ Browser back/forward works perfectly✗ Slightly more complex navigation logic
✓ Deep-linkable: /financials/ledgers?tab=bank_ledger
✓ "View Ledgers" link from PropertyDetail can point to /financials/ledgers?property=X
low effort (≈20 min)

🎯 My recommendation:

Budget: Start with on-the-fly calculation (zero DB changes, ships today). Add a budgets table later when you need real budget editability.

Routing: Switch to sub-routes. It's ~20 minutes, unlocks deep-linking, browser nav, and "View in Financials" links from PropertyDetail.

Charts: The 4 above (Cash Flow line, Budget bar, Maintenance doughnut, Recon doughnut) cover the most important questions with data you already have.
